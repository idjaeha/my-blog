# 블로그 영문 버전 지원

> 최종 수정: 2026-03-18
> 상태: 진행 중

## 목표

모든 한국어 블로그 포스트를 영문으로도 제공하여 국제 독자층을 확보하고, AI 기반 자동 번역과 수동 검토를 결합한 효율적인 다국어 콘텐츠 관리 체계를 구축한다.

## 배경

### 현재 상태

- Astro 6의 네이티브 i18n 지원으로 기본 인프라 80% 구축 완료
- 라우팅: `/blog/post` (한국어), `/en/blog/post` (영문) 구조 완성
- UI 로컬라이제이션: 모든 인터페이스 문구 번역 완료
- SEO 기초: hreflang 태그, 언어별 RSS/sitemap 구현
- 콘텐츠: 한국어 12개, 영문 1개 포스트 (83% 번역 필요)

### 문제점

1. **번역 워크플로우 부재**: 수동 번역만 가능, 자동화 없음
2. **교차 참조 미흡**: 한/영 포스트 간 연결 링크 부재
3. **언어 감지 없음**: 사용자 선호 언어 자동 감지 미지원
4. **번역 상태 추적 불가**: 어떤 포스트가 번역되었는지 파악 어려움

## 접근 방식

### 핵심 전략

- **기존 구조 유지**: Astro i18n 기반 구조는 우수하므로 보강만 진행
- **AI + 휴먼 리뷰**: Claude/GPT-4로 1차 번역 → 수동 검토로 품질 확보
- **점진적 구현**: 인프라 구축 → 테스트 → 전체 콘텐츠 번역

### 주요 설계 결정

#### 1. 번역 메타데이터 추가

```typescript
interface Post {
  translatedFrom?: string; // "ko/original-slug"
  hasTranslations?: string[]; // ["en"]
  translationStatus?: "draft" | "reviewed" | "published";
}
```

- 원본-번역본 관계 명시
- 번역 진행 상태 추적
- 교차 참조 링크 자동 생성 가능

#### 2. MDX 컴포넌트 보존

- Callout, MermaidDiagram 등 커스텀 컴포넌트는 번역 제외
- 컴포넌트 내부 텍스트만 선택적 번역
- Regex 기반 컴포넌트 영역 보호

#### 3. 언어 감지 전략

- 첫 방문: `navigator.language` 감지
- 재방문: localStorage 저장 선호도 우선
- 수동 전환: 선호도 업데이트
- Fallback: 영문 없으면 한국어 + 안내 메시지

## 구현 단계

### 1단계 — 타입 및 인프라 확장

**작업 내용:**

- `src/lib/content/types.ts`: Post 인터페이스에 번역 메타데이터 필드 추가
- `src/lib/content/index.ts`: 번역 정보 로드 로직 추가
- 타입 안전성 검증 (TypeScript 컴파일)

**예상 시간:** 1시간

### 2단계 — AI 번역 명령어 개발

**작업 내용:**

- `.claude/commands/translate-post.md` 생성
  - 대화형 인터페이스 (포스트 목록 표시 및 선택)
  - MDX 파싱 및 frontmatter 추출
  - Claude AI 직접 번역 (별도 API 키 불필요)
  - MDX 컴포넌트 보존 로직
  - 영문 파일 자동 생성 (`src/content/blog/en/`)
- 단일 포스트 테스트 실행
- Slash command로 실행: `/translate-post`

**예상 시간:** 1-2시간

### 3단계 — 교차 참조 링크 구현

**작업 내용:**

- `src/components/BlogPostLayout.astro` 수정
  - 포스트 상단에 언어 전환 버튼 추가
  - `hasTranslations` 메타데이터 기반 조건부 렌더링
  - "Read in English" / "한국어로 읽기" 버튼
  - 스타일링 (기존 디자인 시스템 활용)

**예상 시간:** 1-2시간

### 4단계 — 언어 감지 미들웨어

**작업 내용:**

- `src/middleware/locale-redirect.ts` 생성
  - Browser language detection (`Accept-Language` 헤더)
  - localStorage 기반 선호도 저장/로드
  - 리다이렉트 로직 (조건부)
  - 무한 루프 방지
- `astro.config.ts`에 미들웨어 등록
- 다양한 브라우저 언어 설정 테스트

**예상 시간:** 2-3시간

### 5단계 — SEO 강화

**작업 내용:**

- `src/components/BaseHead.astro` 개선
  - `x-default` hreflang 추가
  - 모든 페이지에 hreflang 적용 확대
- JSON-LD 구조화된 데이터 추가
  - `@type: BlogPosting`
  - `inLanguage` 속성
  - 대체 언어 버전 명시
- Sitemap 검증
  - 모든 locale 포함 확인
  - `<xhtml:link rel="alternate">` 태그

**예상 시간:** 2시간

### 6단계 — 우선순위 포스트 번역

**작업 내용:**

- 인기/최신 포스트 3-5개 선정
- 번역 스크립트 실행
- 수동 검토 및 수정
  - 기술 용어 정확성
  - 자연스러운 표현
  - 코드 블록 검증
- 메타데이터 추가 (translatedFrom, hasTranslations)
- 퍼블리시 및 검증

**예상 시간:** 3-5시간

### 7단계 — 전체 포스트 일괄 번역

**작업 내용:**

- 나머지 포스트 배치 번역 (7-9개)
- 단계별 검토 (하루 2-3개씩)
- 지속적 퍼블리시
- 번역 품질 모니터링
- 피드백 기반 명령어 개선

**예상 시간:** 6-9시간
