# docs/

프로젝트 문서 디렉토리.

## 구조

| 경로               | 설명             | 운영 방식                                                 |
| ------------------ | ---------------- | --------------------------------------------------------- |
| `spec.md`          | 프로젝트 스펙    | 단일 문서를 계속 업데이트, 변경 이력은 git history로 추적 |
| `plans/`           | 작업별 실행 계획 | 작업마다 새 파일 생성 (`YYYY-MM-DD-설명.md`)              |
| `troubleshooting/` | 해결된 이슈 기록 | 이슈마다 새 파일 생성 (`NNN-설명.md`)                     |

`plans/`와 `troubleshooting/` 하위의 `_template.md`에 각각의 작성 포맷이 정의되어 있다.
