블로그 글을 검수합니다.

먼저 `docs/guides/writing-post.md`를 읽고 작성 규칙을 확인한다.

## 대상 글 선택

- `$ARGUMENTS`가 있으면 해당 slug의 글을 대상으로 한다
- 없으면 `src/content/blog/ko/`에서 가장 최근 수정된 .mdx 파일을 대상으로 한다

대상 글의 전체 내용을 읽는다.

## Phase 1: 메타데이터 검증

1. **publishedDate**: ISO 8601 형식에 시분초+타임존(+09:00)이 포함되어 있는지 확인. 오늘 날짜가 아니면 경고
2. **title**: 100자 이하인지 확인
3. **description**: 300자 이하인지 확인
4. **tags**: 비어있지 않은지 확인
5. **category**: til, retrospective, article, tutorial 중 하나인지 확인
6. **series 일관성**: series가 있으면 seriesOrder도 있는지 확인. `mcp__blog-mcp__list-posts`로 같은 시리즈의 다른 글을 조회해서 시리즈 이름 일치 여부와 seriesOrder 연번(1부터 빈 번호 없이) 확인
7. **컴포넌트 import**: 본문에서 `<Callout`, `<LinkCard` 사용 시 해당 import 문이 있는지 확인. MermaidDiagram은 remark 플러그인이 자동 주입하므로 ```mermaid 코드블록 사용 시에는 import 불필요
8. **내부 링크 유효성**: 본문의 `/blog/[slug]` 패턴을 모두 찾아서 `mcp__blog-mcp__get-post`로 존재 여부 확인

## Phase 2: 콘텐츠 리뷰

1. **시리즈 이름 vs 내용 정합성**: 시리즈 이름이 글의 실제 내용을 잘 대표하는지
2. **글 흐름**: 도입부의 주제 설정, 섹션 간 논리적 연결, 결론의 마무리
3. **기술적 정확성**: 코드 예시와 설명의 일치, API/라이브러리 이름 정확성
4. **독자 관점**: 사전 지식 없이 따라갈 수 있는지, 전문 용어 설명 여부
5. **시리즈 연결**: 이전/다음 글 참조 정확성, 예고한 내용과 실제 내용 일치 여부
6. **한국어 표현**: 오타, 비문, 어색한 표현

## 출력 형식

아래 형식으로 리뷰 결과를 출력한다:

```
## 리뷰 결과: [slug]

### 1. 메타데이터 검증
- [x/!] 각 항목의 통과/실패 여부와 상세 내용

### 2. 콘텐츠 리뷰
- 각 항목별 피드백

### 요약
- 주요 문제점과 개선 제안
```

대상 slug: $ARGUMENTS
